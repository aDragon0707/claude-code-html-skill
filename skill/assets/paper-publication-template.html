1Introduction

Open with the problem in plain language, then connect it to the technical question. Preserve the source's scope and level of certainty rather than manufacturing a dramatic hook.

This paragraph should state the paper's central claim and the type of evidence used to support it. Place citations beside the claim they support [1, 2]. Keep the reading column calm; let figures and section breaks create rhythm.

Explain why the result matters, but distinguish measured findings from interpretation. If the source uses a proxy, assumption, or theoretical model, state that boundary here rather than hiding it in the appendix.

1.1Research question

Define the specific question and the falsifiable distinction the paper tests. Preserve important negative claims and alternatives instead of presenting only the preferred explanation.

2Methods

Describe the method at the level needed to interpret the results. Keep exact dataset names, hardware or model settings, units, preprocessing, and exclusion rules from the source.

score(x) = Σᵢ weightᵢ · featureᵢ(x)

2.1Measurement

Transcribe equations and table values carefully. If a formula was recovered by OCR or manually rebuilt, label it for verification.

3Results

Lead with the measured result, then explain it. Use consistent semantic colors across every figure so the same concept does not change identity from section to section.

SOURCE RESULT

Primary finding

State the result with the metric, comparison, sample, and uncertainty.

DERIVED INTERPRETATION

What it suggests

Keep interpretation separate from the source measurement.

4Mechanism

Use a diagram when the claim depends on layer, sequence, hierarchy, or flow. Put the diagram next to the prose that interprets it.

5Limitations

Keep limitations in the main reading path. Do not collapse or visually minimize evidence that constrains the headline conclusion.

Evidence boundary

List missing controls, proxy assumptions, extraction uncertainty, external-validity limits, and checks that still require independent verification.

6Discussion

Separate what the evidence supports from open questions and recommendations. If the HTML adds a new calculation or synthesis, label it as derived rather than attributing it to the paper.
